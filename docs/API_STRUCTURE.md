## 🌐 API Overview

### Base URL

Development:
```
http://localhost:8080
```

Production:
```
https://api.scylla.com
```

### API Versioning

Semua API menggunakan version prefix:
```
/v1/{resource}
```

### Content Type

Semua request dan response menggunakan:
```
Content-Type: application/json
```

## 📤 Request/Response Format

### Request Format

#### Headers
```http
Content-Type: application/json
Accept-Language: en|id (optional)
X-Request-ID: (auto-generated by middleware)
```

#### Body
```json
{
  "field1": "value1",
  "field2": "value2"
}
```

### Response Format

#### Success Response
```json
{
  "message": "Success",
  "data": {
    "id": "123",
    "name": "Example"
  },
  "errors": null,
  "paging": null,
  "request_id": "507f1f77bcf86cd799439011"
}
```

#### Error Response
```json
{
  "message": "Bad Request",
  "data": null,
  "errors": [
    {
      "field": "job_name",
      "message": "job_name is required"
    }
  ],
  "request_id": "507f1f77bcf86cd799439011"
}
```

#### List Response with Pagination
```json
{
  "message": "Success",
  "data": [
    {
      "id": "123",
      "name": "Item 1"
    },
    {
      "id": "124",
      "name": "Item 2"
    }
  ],
  "errors": null,
  "paging": {
    "total_record": 100,
    "page_current": 1,
    "page_limit": 10,
    "page_total": 10
  },
  "request_id": "507f1f77bcf86cd799439011"
}
```

## ⚠️ Error Handling

### HTTP Status Codes

- `200 OK`: Success
- `201 Created`: Resource created successfully
- `400 Bad Request`: Invalid request data
- `401 Unauthorized`: Authentication required
- `403 Forbidden`: Access denied
- `404 Not Found`: Resource not found
- `422 Unprocessable Entity`: Validation error
- `500 Internal Server Error`: Server error

### Error Response Structure

```json
{
  "message": "Error message",
  "data": null,
  "errors": [
    {
      "field": "field_name",
      "message": "Error message for this field"
    }
  ],
  "request_id": "507f1f77bcf86cd799439011"
}
```

### Common Error Scenarios

#### Validation Error
```json
{
  "message": "Bad Request",
  "errors": [
    {
      "field": "email",
      "message": "email must be a valid email address"
    },
    {
      "field": "password",
      "message": "password is required"
    }
  ]
}
```

#### Not Found Error
```json
{
  "message": "Not found",
  "data": null,
  "errors": null
}
```

#### Server Error
```json
{
  "message": "Internal server error",
  "data": null,
  "errors": null
}
```

## 📄 Pagination

### Query Parameters

- `page`: Halaman (default: 1)
- `limit`: Jumlah data per halaman (default: 10)
- `sort`: Sorting (format: `field:ASC` atau `field:DESC`)
- `active`: Filter active status (0=all, 1=active, 2=inactive)
- `query`: Search query (optional)

### Example Request

```http
GET /v1/jobs?page=1&limit=10&sort=created_at:DESC&active=1&query=test
```

### Pagination Response

```json
{
  "paging": {
    "total_record": 100,
    "page_current": 1,
    "page_limit": 10,
    "page_total": 10
  }
}
```

### Sorting

Multiple field sorting:
```
sort=created_at:DESC,updated_at:ASC
```

## 📝 Request Validation

### Validation Rules

Validation menggunakan struct tags:

```go
type CreateJobBody struct {
    JobName string `json:"job_name" validate:"required,max=100"`
    JobDesc string `json:"job_desc" validate:"required,max=255"`
    JobType string `json:"job_type" validate:"required,oneof=duration cron daily weekly monthly one_time"`
    Email   string `json:"email" validate:"required,email"`
    Age     int    `json:"age" validate:"required,min=18,max=100"`
}
```

### Common Validation Tags

- `required`: Field wajib diisi
- `max=n`: Maximum length/value
- `min=n`: Minimum length/value
- `email`: Valid email format
- `oneof=val1 val2`: Must be one of the values
- `numeric`: Must be numeric
- `alphanum`: Alphanumeric only
