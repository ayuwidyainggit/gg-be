## Goal

Perbaiki regresi Backend Scylla X untuk Secondary Sales Report di modul `sales`:

- SX-2174: field `qty`, `qty_return`, `return_rate`, dan `net_sales_return` pada `GET /sales/v1/reports/secondary-sales/sum-date` harus berasal dari query transaksional order/return sesuai SQL acuan Jira.
- SX-2182: pastikan export `POST /sales/v1/reports/secondary-sales` tetap mendukung `cust_id` string dan array, multi business unit yang authorized, filter array aman, dan metadata resolve memakai row-level `cust_id`.

## Non-goals

- Tidak mengubah route endpoint atau nama field response existing.
- Tidak mengubah FE behavior Select All.
- Tidak mengubah schema DB atau membuat migration.
- Tidak menyentuh service selain `sales` kecuali implementer menemukan regresi SX-2182 masih terkait `master` business-unit; discovery saat ini menunjukkan fokus cukup di `sales`.
- Tidak menyimpan, paste, atau commit Authorization token Jira/cURL.
- Tidak mengganti desain async export/RMQ selain payload/filter compatibility yang sudah ada.

## Requirements

1. `sum-date` harus tetap menerima `month`, optional `year`, dan optional `cust_id` dengan authorization existing.
2. `month=6&year=2026` harus memakai date range `2026-06-01 00:00:00` sampai sebelum `2026-07-01 00:00:00`, atau inclusive equivalent yang tidak bocor ke bulan berikutnya.
3. `qty` harus dari `Number of Product Sold`: `SUM((qty3 * conv_unit2 * conv_unit3) + (qty2 * conv_unit2) + qty1)` dari `sls.order` + `sls.order_detail`.
4. `qty_return` harus dari `Number of Product Return` dengan formula satuan terkecil yang sama dari `sls.return_det` + `sls.return` + linked `sls.order`.
5. `net_sales_return` harus dari `Return Value`: net sales include PPN return.
6. `return_rate` harus dari `Return Rate (%)`: `ROUND((return value / order net sales include PPN) * 100, 2)`, safe zero saat order net sales `0`.
7. Summary field lain (`total_gross_sale`, `total_discount_promo`, `total_ppn`, `net_sales_exc_ppn`, `net_sales`) tidak boleh regress dari mapping docs/latest repo behavior.
8. `cust_id` export harus menerima legacy string, array string, missing, empty string, dan empty array sesuai behavior existing.
9. Unauthorized requested `cust_id` harus return `403`, bukan silent ignore.
10. Empty optional filters `distributor_ids`, `outlet_ids`, `salesman_ids`, `pro_ids` berarti no filter.
11. Banyak optional filter IDs tidak boleh membuat result kosong bila data ada dan IDs valid.
12. Export metadata product/outlet/salesman/distributor harus tetap memakai row-level `cust_id`, dengan parent product fallback bila dibutuhkan.
13. `report.list.cust_id` harus tetap auth owner, bukan child `cust_id` pertama.

## Acceptance Criteria

SX-2174:

- `GET /sales/v1/reports/secondary-sales/sum-date?month=6&year=2026&cust_id=C260020001` mengembalikan `qty`, `qty_return`, `return_rate`, dan `net_sales_return` yang sama dengan SQL acuan Jira untuk periode Juni 2026.
- `qty` dan `qty_return` memakai formula satuan terkecil, bukan penjumlahan mentah `qty1 + qty2 + qty3`.
- `return_rate` adalah value-based `return/order * 100`, rounded 2 decimals, bukan qty-based ratio.
- `net_sales_return` adalah return include PPN, bukan exclude PPN.
- No data/null menghasilkan `0`, bukan `null` atau error.
- Existing field response tetap backward-compatible.

SX-2182:

- Export menerima `cust_id: "C260020001"` dan `cust_id: ["C260020001"]`.
- Missing/empty `cust_id` fallback ke auth `cust_id`.
- Multi authorized `cust_id` menghasilkan data gabungan.
- Unauthorized `cust_id` menghasilkan `403` dan tidak membuat job export.
- Empty optional filter arrays tidak menambah predicate yang memfilter semua row.
- Filter banyak IDs tetap memakai parameter binding dan tidak menghasilkan SQL `IN ()`.
- Export selesai dengan `file_url` populated ketika runtime/storage tersedia.
- `report.list.cust_id` tetap auth owner.
- Metadata export benar untuk row child `cust_id`.

## Decisions/Assumptions

Keputusan:

- Ikuti SQL acuan Jira: `return_rate = return net sales include PPN / order net sales include PPN * 100`.
- `net_sales_return` memakai return include PPN.
- `qty` dan `qty_return` memakai conversion formula satuan terkecil.
- Untuk SX-2182, current implementation sudah sebagian besar memenuhi requirement; implementer fokus regression test dan manual validation, bukan rewrite besar.
- Missing/empty `cust_id` tetap fallback ke auth cust, bukan otomatis semua child.

Assumptions / Open Questions:

- Assumption slice-safe: `authCustID == parentCustID` adalah principal; `authCustID != parentCustID` adalah distributor, sesuai resolver existing.
- Assumption slice-safe: FE tidak membutuhkan optional summary filters baru selain yang sudah ada kecuali prompt/Jira benar-benar mengirim `outlet_ids`, `salesman_ids`, atau `product_ids/pro_ids` ke `sum-date`.

Question gate: tidak ada pertanyaan blocking; requirement cukup untuk plan siap implementasi. Pertanyaan optional di atas bisa dijawab lewat validasi runtime/QA.
