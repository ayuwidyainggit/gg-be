# Plan Audit SX-2131 / SX-2184 — Sales Order Purchase Order / Taking Order

Readiness: `ready-for-implementation`
Plan Quality Gate: `PASS_FOR_SLICE`

## Goal

Audit status implementasi `order_type = "O"` untuk `POST /sales/v1/orders` di service `sales`, pastikan acceptance criteria SX-2131/SX-2184 sudah tertutup, dan berikan handoff jika masih perlu eksekusi lanjutan.

## Non-goals

- Tidak mengubah source code, test, migration, package, lockfile, env, atau aset dari planner ini.
- Tidak memperbaiki formula stock validation `SO` yang tampak aneh di Jira; itu di luar scope kecuali tiket baru eksplisit.
- Tidak mengubah behavior `SO`, `C`, nil, atau empty `order_type` selain memastikan tetap as-is.
- Tidak menyalin credential, token, header auth, atau isi `.env` ke artefak.

## Scope

- Module target: `sales`.
- Endpoint target: `POST /sales/v1/orders`.
- Area audit: controller create, validate-order service, order store service, order type helper, model/entity, migration, dan test SX-2184.
- Artefak utama: plan ini sebagai source of truth untuk audit/handoff.

## Requirements

- API menerima optional `order_type` dengan value `O`, `C`, `SO`; missing/empty tidak breaking.
- `order_type = "O"` harus skip stock validation blocker, tetapi non-stock validations tetap berjalan.
- `order_type = "O"` tidak boleh memanggil inventory mutation create.
- `order_type = "O"` menyimpan `sls.order.order_type = O`, fallback `opr_type = O`, `validate_stok = false`, `validate_stok_message = NULL`.
- `order_type = "O"` menyimpan `qty_po1/2/3` dan `original_qty_po1/2/3` dari qty original request.
- `SO`, `C`, nil, dan empty harus tetap memakai flow stock validation/mutation existing.
- Migration additive dan rollback tersedia.

## Acceptance Criteria

- `sales/controller/order_controller.go` mengarahkan `O` ke non-stock validation path, bukan full bypass semua validation.
- `sales/service/validate_order_service.go` menyediakan path yang skip hanya stock query/loop dan tetap menghitung credit limit, overdue, outstanding.
- `sales/service/order_service.go` tidak membuat `SalesOrderStockUpdate` untuk `O`.
- `sales/service/order_service.go` tetap membuat stock update untuk nil/empty/`SO`/`C` ketika status existing memproses mutasi.
- `sales/migration/sls.order/add_order_type_and_original_qty_po_fields.sql` dan rollback tersedia.
- Targeted SX-2184 controller/service tests pass.
- Full `sales` test suite pass.

- Hanya `strings.TrimSpace(order_type) == "O"` yang boleh skip stock validation blocker dan inventory mutation create.

- Non-`O` tidak boleh masuk branch Taking Order.
- `original_qty_po*` harus dari qty original payload, bukan qty hasil penyesuaian stock.
- Create Taking Order awal tidak boleh insert `inv.stock` atau update `inv.warehouse_stock`.
- Semua write tetap transaction-aware.
- Planner ini hanya menulis artefak `.opencode/`, bukan source.
