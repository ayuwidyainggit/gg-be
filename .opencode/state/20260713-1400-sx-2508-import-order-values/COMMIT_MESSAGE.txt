fix(sales): SX-2508 persist import slot fields and remap DetailV2 quantity orientation

Imported sales orders now persist the slot promo fields (PromoSo1,
PromoFinal1) and the secondary/tertiary system prices derived from the
parent SellPrice3, while keeping QtyPo explicit-nil.  DetailV2 respects
the new is_sales_mapping flag: mapped orders keep the stored
{Qty1,Qty2,Qty3} / {Qty1Final,Qty2Final,Qty3Final} without re-running
the global converter; non-mapped orders still use ConvToQtyConversion
but the converter output is now remapped to the API orientation
Large/Middle/Small (Qty1=qtyConversion.Qty3, Qty2=qtyConversion.Qty2,
Qty3=qtyConversion.Qty1).

sales/service/order_stock_helper.go is unchanged.
