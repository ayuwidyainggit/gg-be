```sql
FROM sls."order"
WHERE ro_no='SO2606180003' AND cust_id='C260020001';
```

Result:

```text

```

Stock rows:

```sql
SELECT cust_id, stock_id, stock_date, tr_code, tr_no, wh_id, pro_id, item_cdn,
       qty_in, qty_out, qty_in_order, qty_out_order, ref_det_id, created_at
FROM inv.stock
WHERE cust_id='C260020001'
  AND tr_no IN ('SO2606180003','SO2606180003-CO')
ORDER BY stock_id;
```

Result:

```text
0 rows
```

Expected for this local case: no stock rows because `Need Review` order had no source `inv.stock` movement. Fix skips stock write and still updates order status.

Warehouse stock snapshot:


No warehouse stock mutation occurred during cancel, as expected for no source stock movement.
